Step 01

Tokenization

🔊
Before a language model can understand your text, it needs to convert it into numbers. This process is called tokenization — your sentence is broken into small pieces called tokens, each assigned a unique ID number.

What is Tokenization?

Before the model can process text, it needs to convert words into numbers. Think of it like a dictionary: every word (or sub-word) gets a unique ID number. The model only understands numbers, never raw text!

Key Points
  • Common words = 1 token. Rare words are split into sub-words
  • GPT-4 uses ~100,000 unique token IDs (BPE vocabulary)
  • Spaces and punctuation are often merged into adjacent tokens
  • The same text always maps to the same token sequence
Step 02

Embeddings & Positional Encoding

🔊
A token ID is just a number with no meaning. An embedding converts it into a vector — a list of numbers that capture semantic meaning. Similar words end up with similar vectors. We also add positional encoding using sine/cosine waves so the model knows word order.

💡
What is an Embedding?

A token ID (like 7592) is just a number — it has no meaning by itself. An embedding converts it into a vector (a list of numbers). These numbers capture meaning: similar words end up with similar vectors!

Why Position Info?

Without position info, "dog bites man" and "man bites dog" look identical! We add sine/cosine waves at different frequencies — one unique signature per position.

REAL-WORLD ANALOGY

Step 03

Self-Attention (Q, K, V)

🔊
Self-attention lets every word look at every other word. For each token we create 3 vectors — Q (what I'm looking for), K (what I have), V (my information). Scores are computed as Q·Kᵀ/√d, then softmax gives attention weights — how much each word attends to others.
↑ Click to replay audio
💡
What is Self-Attention?

Self-attention lets each word look at all other words and decide how much to "pay attention" to them. Formula: Attention = softmax(QKᵀ / √d_k) · V

We create 3 copies of each token's vector: Q (what am I looking for?), K (what do I have?), V (what info do I share?).

STEP 2: Compute Scores — Q · Kᵀ / √d
STEP 3: Apply Softmax → Attention Weights (0 to 1, sum=1)

Step 04

Add & Norm (Residual Connection)

🔊
After attention, we add the original input back (residual connection) to prevent information loss and help gradients flow during training. Then Layer Norm rescales values to mean=0, std=1 — keeping training stable across hundreds of layers.

💡
What is Add & Norm?

After attention, we do two things:
1. Add: we add the original input back to the attention output — called a "residual connection". This preserves information and prevents the signal from disappearing.
2. Norm: Layer Normalization rescales values to have mean=0 and std=1, keeping training stable.

Why Residual Connections?

In deep networks, gradients can vanish during training. Adding the original input back (skip connection) gives gradients a "highway" to flow through — the same idea as ResNet in image recognition.

REAL-WORLD ANALOGY

Step 05

Feed Forward Network

🔊
After attention gathers context across tokens, the Feed Forward Network processes each token independently. It expands the dimension 4×, applies ReLU (zeroing negatives), then compresses back. Most of the model's factual knowledge lives here.
↑ Click to replay audio
💡
What is the Feed Forward Layer?

After attention gathers context, the Feed Forward Network (FFN) processes each position independently. It's a 2-layer neural network: first it expands the dimension (×4), applies ReLU activation, then compresses back down.

Formula: FFN(x) = max(0, xW₁ + b₁)W₂ + b₂

Negative values are zeroed out. This introduces non-linearity, letting the model learn complex patterns.
🔑
Why FFN After Attention?
  • Attention mixes information across positions. FFN processes each position independently
  • The 4× expansion creates room for richer representations
  • FFN layers hold most of the model's "factual knowledge"
  • In GPT-3: input dim=12288, hidden dim=49152 (4× larger!)
Step 06

Backpropagation & Learning

🔊
Forward pass: data flows through all layers to produce a prediction. Backward pass: we measure the error (loss), then use the chain rule to compute how each weight should change. The optimizer updates all weights slightly. Repeat for billions of tokens — that's training!

💡
How Does the Model Learn?

Training has two phases: Forward pass (compute prediction) and Backward pass (compute how wrong we were, then update weights). The backward pass uses the chain rule of calculus to propagate the error back through every layer.

Key Concepts
  • Loss function: measures how wrong the prediction is (Cross-Entropy)
  • Gradient: the direction & size to adjust each weight
  • Learning rate: how big each weight update step is (e.g. 0.0001)
  • Optimizer: Adam adjusts learning rates per parameter automatically
  • GPT-3 trained on 300 billion tokens — billions of these update steps!
🎯
REAL-WORLD ANALOGY

Imagine throwing a ball at a target blindfolded. After each throw, someone tells you how far off you were and in which direction. You adjust your next throw a little. After millions of throws, you get very accurate. Backpropagation is exactly this feedback loop — but for every weight in the network simultaneously.

Step 07

Context Window & Temperature

🔊
Context Window = the model's short-term memory. Tokens beyond it are forgotten. Temperature controls creativity: low (0.1) = always picks the most likely word — precise; high (0.9) = samples less likely words — creative. Use the sliders to see how both parameters change model behavior.
